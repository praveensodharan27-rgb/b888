# Fix: Next.js chunk 404 errors (main-app.js, page.js, layout.css, etc.)

## What the error means

The browser loads the page HTML, then requests JavaScript/CSS chunks (e.g. `main-app.js`, `app-pages-internals.js`, `page.js`, `layout.css`). The **server returns 404** for those URLs, so the app can’t run or style correctly.

Those files are generated by Next.js and served from `/_next/static/...`. A 404 means either:

- The chunk **path/hash changed** (e.g. after a restart or rebuild) but the browser is still using **old references** from cache.
- The **.next** build output is stale or corrupted, so the dev server doesn’t have the files the HTML is asking for.

## Fix (do in order)

### 1. Stop the dev server

In the terminal where `npm run dev` is running, press **Ctrl+C**.

### 2. Delete the Next.js cache

From the **frontend** folder run:

```bash
# Windows (PowerShell)
Remove-Item -Recurse -Force .next

# Or manually delete the folder: frontend\.next
```

### 3. Restart the dev server

```bash
cd frontend
npm run dev
```

Wait until you see “Ready in …” and the local URL (e.g. `http://localhost:3000`).

### 4. Hard refresh the browser

- **Chrome/Edge:** `Ctrl + Shift + R` or open DevTools (F12) → right‑click Refresh → **Empty Cache and Hard Reload**.
- Or clear site data for `localhost:3000`: Application → Storage → Clear site data.

### 5. Open the app on the same host

Use the **exact** URL shown by the dev server, e.g.:

- `http://localhost:3000`

Avoid mixing `http://127.0.0.1:3000` and `http://localhost:3000` if you had 404s (stick to one).

## If it still happens

- Make sure you’re not running **production** (`npm run start`) with an old or incomplete `.next` folder. For production, run `npm run build` then `npm run start`.
- Disable browser extensions for localhost (they can block or alter requests).
- Try an incognito/private window to rule out extensions and cache.

## Prevention

- After pulling big changes or switching branches, run steps 1–3 (stop dev, delete `.next`, start dev again) and then hard refresh.
- Use one consistent origin (e.g. always `http://localhost:3000`) when developing.
